# Lab 7.4: Troubleshoot a Network Issue

**Role:** IT Help Desk

## 🎯 Objective
To diagnose hardware-related network connectivity problems and manage the incident resolution process through an IT ticketing system.

## 🚀 Execution Steps
1. **Hardware Diagnosis:** 
   * Investigated the missing Wi-Fi icon on the user's laptop. Identified that the physical wireless switch on the exterior of the laptop was turned off, disabling the network adapter.
2. **Issue Resolution:** 
   * Turned on the physical Wi-Fi switch to restore the hardware functionality and verify network connectivity.
3. **Ticketing System Management:** 
   * Managed the issue via the Help Desk ticketing system. Opened a ticket, documented the root cause (hardware switch disabled) and the troubleshooting steps, and formally closed the ticket upon successful resolution.

## 📊 Results & Evidence
* Successfully restored Wi-Fi connectivity by resolving the physical hardware issue.
* Completed the support lifecycle by accurately documenting and closing the help desk ticket.

## 🧠 Key Takeaways
* **Hardware Verification First:** Learned that software symptoms, like a missing Wi-Fi icon, can often be caused by simple physical hardware issues, such as an external Wi-Fi switch being turned off on a laptop.
* **Help Desk Workflow:** Gained practical experience in incident management. Proper documentation (opening a ticket, recording the problem, noting the solution, and closing it) is crucial for tracking issues and maintaining a clear history for the IT support team.
